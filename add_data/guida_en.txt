[H1]DataFlow Procurement Software User Guide
[INFO]Documentation version: November 2025

This guide will help you use DataFlow, the software for managing Requests for Offer.

[H1]0. Getting Started

[H2]Welcome to DataFlow
DataFlow is a desktop application designed to simplify the management of Requests for Offer (RFQ) in your procurement process.

With DataFlow you can:
[LIST] • Create and manage RFQs with ease
[LIST] • Compare quotes from multiple suppliers simultaneously
[LIST] • Attach documents and offers to each request
[LIST] • Keep a complete record of RFQ history
[LIST] • Export data to Excel for further analysis

[H2]First Launch
On first launch, you will be asked to enter your [BOLD]First Name [NORMAL]and [BOLD]Last Name[NORMAL].
[LIST] • Enter your first and last name in the appropriate fields
[LIST] • The system will automatically generate your personal username (first letter of name + last name, all lowercase and without accents)
[LIST] • This username will be associated with the RFQs you create and can be used to filter requests

Subsequently, you will be shown the End User License Agreement window.
[LIST] • Read the Freeware license terms carefully
[LIST] • Click 'Accept' to continue or 'Exit' to close the program

The program will automatically create a user-specific database in the folder:
C:\Users\[username]\Documents\DataFlow_[username]\Database\dataflow_db_[username].db

You don't need to do anything, everything is configured automatically. All documents (database and any attachments) can be found in your personalized DataFlow folder.

[H2]SQDC Analysis
The SQDC analysis is a tool to evaluate and compare suppliers based on four fundamental criteria: [BOLD]Safety[BOLD], [BOLD]Quality[BOLD], [BOLD]Delivery[BOLD], and [BOLD]Cost[BOLD].

The '[CODE]... SQDC[CODE]' button in the RFQ control panel allows you to create or open an SQDC analysis for that specific request. The button text changes dynamically:
[LIST] • [BOLD]Create SQDC Analysis:[NORMAL] When no analysis has been saved yet for this RFQ
[LIST] • [BOLD]Open SQDC Analysis:[NORMAL] When an SQDC analysis has already been saved

How to use the SQDC analysis:

[BOLD]1. [NORMAL]Weights Tab (%)
[LIST]   • Assign a percentage weight to each of the four criteria (Safety, Quality, Delivery, Cost)
[LIST]   • The sum of weights must be exactly 100%
[LIST]   • Weights represent the relative importance of each criterion in the final evaluation
[LIST]   • Each criterion has a description explaining what it represents

[BOLD]2. [NORMAL]Scores Tab (1-10)
[LIST]   • For each supplier, assign a score from 1 to 10 for each of the four criteria
[LIST]   • Score 1 represents the worst performance, 10 the best
[LIST]   • Scores are multiplied by their respective weights and summed to obtain the total score
[LIST]   • The supplier with the highest total score is the best according to the analysis

[BOLD]3. [NORMAL]Automatic Calculation
[LIST]   • The system automatically calculates the total score for each supplier
[LIST]   • Total scores are displayed in real time as you enter scores
[LIST]   • You cannot switch to the Scores tab if weights do not sum to 100%

[H3]Automatic Cost Score Calculation

The system automatically calculates the Cost score for each supplier based on the economic competitiveness of the offer:

[BOLD]Calculation formula:[BOLD]
[LIST]1. The system identifies the supplier with the lowest total price → automatically receives score [BOLD]10[BOLD]
[LIST]2. For other suppliers, the score is calculated with this formula:
[CODE]Cost Score = 10 × (Minimum Price / Supplier Price)[CODE]
[LIST]3. The score is rounded to the nearest integer

[BOLD]Practical example:[BOLD]
[LIST] • Supplier A: Total price = €1000 → Cost Score = [BOLD]10[BOLD] (lowest price)
[LIST] • Supplier B: Total price = €1200 → Cost Score = 10 × (1000/1200) = 8.33 → [BOLD]8[BOLD]
[LIST] • Supplier C: Total price = €1500 → Cost Score = 10 × (1000/1500) = 6.67 → [BOLD]7[BOLD]

[WARNING_RED]IMPORTANT: Requirements for automatic Cost calculation

The automatic Cost calculation verifies that each supplier has:
[LIST] • A [BOLD]valid numeric[BOLD] price for [BOLD]ALL[BOLD] items in the RFQ
[LIST] • A [BOLD]valid numeric[BOLD] quantity for [BOLD]ALL[BOLD] items

[BOLD]If a supplier has even one price:[BOLD]
[LIST] • Empty (uncompleted cell)
[LIST] • Marked as text: "X", "ND", "N/A", "N.D.", "-", etc.
[LIST] • With missing or non-numeric quantity

→ Their Cost score is automatically set to [BOLD]0[BOLD] and a [BOLD]⚠️ yellow warning[BOLD] will appear in the score field.

[BOLD]Why this rule?[BOLD]
It ensures that only suppliers who have submitted a complete offer are evaluated fairly. A supplier with partial pricing cannot be correctly compared with others in terms of economic competitiveness.

[INFO]After automatic calculation, you can still [BOLD]manually modify[BOLD] the Cost score if needed (for example, to give a partial score to a supplier for qualitative evaluation or to handle special cases).

[BOLD]4. [NORMAL]Excel Export
[LIST]   • Click '[CODE]📊 Export Excel[CODE]' to generate an Excel file with the complete analysis
[LIST]   • The file includes all data: weights, scores for each supplier, and total scores
[LIST]   • The supplier with the highest score is highlighted in green
[LIST]   • The template used depends on the language selected in settings

[BOLD]5. [NORMAL]Saving
[LIST]   • Click '[CODE]💾 Save SQDC[CODE]' to save the analysis as an Internal Document
[LIST]   • The saved analysis can be reopened at any time by clicking the SQDC button

[H2]Database Management
[LIST] • [BOLD]Migrate Database:[NORMAL] In the program installation folder (C:\Program Files\WindowsApps\...) you will find a tool "Database Migration Tool.exe" to import data from a database created with a previous version of the program (SQLite .db) to the new user-specific DB database (dataflow_db_[username].db) currently in use.

When to use migration: [BOLD]
If you are updating DataFlow from version 1.x to version 2.0.0 (or later) and already have data saved in the previous version. The migration automatically converts the old SQLite database to the new DB format, which is more performant and modern.

What gets migrated: [BOLD]
All RFQs (complete with all data), suppliers, items, prices, attachments, notes and all associated information are transferred to the new database without loss.

Step-by-step procedure:
[BOLD]1. [NORMAL]Close DataFlow if it is open
[BOLD]2. [NORMAL]Make sure you have a backup of the current database
[BOLD]3. [NORMAL]In the program installation folder, launch "Database Migration Tool.exe"
[BOLD]4. [NORMAL]Click the [CODE]📂 Select Old Database[CODE] button and choose the SQLite database file from the previous version (typically [CODE]gestione_offerte.db[CODE])
[BOLD]5. [NORMAL]Click the [CODE]📂 Select Config.ini[CODE] button and choose the [CODE]config.ini[CODE] file of DataFlow 2.0.0 (located in C:\Users\[username]\AppData\Local\DataFlow)
[BOLD]6. [NORMAL]Click [CODE]🔄 Start Migration[CODE] to start the conversion
[BOLD]7. [NORMAL]The tool will automatically copy all data to the new DB format in the Documents\DataFlow folder
[BOLD]8. [NORMAL]When complete, restart DataFlow and verify that the data has been migrated correctly by opening some RFQs

[BOLD]IMPORTANT: [WARNING]The operation is safe and does not modify the original database. All data is copied to the new database maintaining the correct format. The tool also extracts attachments saved as BLOBs in the old database and saves them as files in the Attachments folder.

[H1]5. Common Problems and Solutions
This section will help you resolve the most common problems you might encounter while using DataFlow.

[H2]Grid Changes Not Saving

[BOLD]Symptom: [NORMAL]I modified a cell in the grid but reopening it the data is as before.

[BOLD]Cause: [NORMAL]You probably used the context menu (right click) instead of editing directly.

[BOLD]Solution:
[LIST]1. To modify the [BOLD]content [NORMAL]of cells (items/prices): [BOLD]double click [NORMAL]on the cell, edit, press Enter
[LIST]2. To [BOLD]add [NORMAL]items: use the [CODE]➕ Add Item [NORMAL]button
[LIST]3. To [BOLD]delete [NORMAL]items: select the row and use [CODE]🗑 Remove Selected Item

[WARNING]The right-click menu (Insert row, Delete row, etc.) is only for temporary display and [BOLD]does not save anything to the database[NORMAL].

[H2]Database Locked
Error message: [BOLD]'The database is in use' or 'database is locked'

Cause: [BOLD]Another instance of the program is already using the database.

Solution:
[BOLD]1. [NORMAL]Check all open DataFlow windows
[BOLD]2. [NORMAL]Close any other instances of the program
[BOLD]3. [NORMAL]If you use the database on a network, verify that no other user is using it
[BOLD]4. [NORMAL]Restart the program

[BOLD]ATTENTION: [WARNING_RED]NEVER open the same database for writing from multiple workstations simultaneously. It will cause permanent data corruption![/WARNING_RED]

[H2]Excel Import Errors
Error message: [BOLD]'Excel file read error' or 'Missing columns'

Possible causes and solutions:
[LIST] • [BOLD]Wrong file format:[NORMAL] Verify that you are using an .xlsx file (Excel 2007 or later)
[LIST] • [BOLD]Missing columns:[NORMAL] Check that all required columns are present and in the correct order (see 'Import from Excel' section)
[LIST] • [BOLD]Wrong RFQ Type:[NORMAL] If you selected 'Full Supply' but the file has 7 columns (or vice versa), change the RFQ type before importing
[LIST] • [BOLD]Corrupted file:[NORMAL] Try opening the file in Excel and saving it again
[LIST] • [BOLD]Invalid data:[NORMAL] Verify that cells contain the correct data types (text in text columns, numbers in quantity columns)

Tip: [BOLD]Use the template files available in the [CODE]add_data[CODE] folder as a reference for the correct structure.

[H2]Attachments Not Found
Error message: [BOLD]'Invalid path' or 'File not found'

Cause: [BOLD]The DataFlow folder (containing the attachments archive) has been moved or deleted.

Solution:
[BOLD]1. [NORMAL]Go to Settings > Database
[BOLD]2. [NORMAL]Click '[CODE]📁 Change DataFlow Folder...[CODE]'
[BOLD]3. [NORMAL]Select the new location of the DataFlow folder (which includes the Attachments folder with all attachments)
[BOLD]4. [NORMAL]The program will automatically update all references

Prevention: [BOLD]If you need to move the complete DataFlow structure (database and attachments), ALWAYS do it through the 'Change DataFlow Folder' function in Settings, never manually.

[H2]Recovery from Backup
If you have lost data or the database is corrupted, you can recover from a backup.

Procedure:
[BOLD]1. [NORMAL]Close DataFlow
[BOLD]2. [NORMAL]Go to the folder where you saved the backups (default in folder C:\Users\[username]\Documents\DataFlow\Database)
[BOLD]3. [NORMAL]Find the most recent backup (files have names like [CODE]dataflow_db_[username]_backup_YYYYMMDD_HHMMSS.db[CODE])
[BOLD]4. [NORMAL]Make a copy of the backup with a different name (e.g. [CODE]dataflow_db_[username]_recupero.db[CODE])
[BOLD]5. [NORMAL]Delete or rename the corrupted database [CODE]dataflow_db_[username].db[CODE]
[BOLD]6. [NORMAL]Rename the backup file to [CODE]dataflow_db_[username].db[CODE]
[BOLD]7. [NORMAL]Restart DataFlow

[BOLD]IMPORTANT: [WARNING]This procedure replaces the current database with the backup. Data created after the backup will be lost.

[H1]7. Glossary
This section defines the technical terms used in DataFlow.

[LIST] • [BOLD]RFQ:[NORMAL] Request for Offer. Document requesting a quote from one or more suppliers for materials or services.
[LIST] • [BOLD]Full Supply:[NORMAL] RFQ type for standard purchase of complete materials supplied by the seller.
[LIST] • [BOLD]Processing (P/L):[NORMAL] RFQ type for external processing on materials provided by the client to the supplier.
[LIST] • [BOLD]Raw:[NORMAL] Unprocessed material provided for external processing (e.g. raw sheet metal to be stamped).
[LIST] • [BOLD]Archive:[NORMAL] State of a completed or no longer active RFQ, moved to the 'Archived RFQs' tab.
[LIST] • [BOLD]External Link:[NORMAL] Attachment saved as a separate file in the Attachments folder, not stored in the database.
[LIST] • [BOLD]DB:[NORMAL] High-performance analytical database system used by DataFlow 2.0.0 to manage RFQ data.
[LIST] • [BOLD]Username:[NORMAL] Unique identifier automatically generated at DataFlow's first launch by concatenating the first letter of the first name with the full last name, all in lowercase without spaces and accents. Example: John Smith → [CODE]jsmith[CODE], Mary Brown → [CODE]mbrown[CODE]. The username is used to name the user folder ([CODE]DataFlow_[username][CODE]), the personal database ([CODE]dataflow_db_[username].db[CODE]), and to filter RFQs in multi-user environments. The username is automatically saved in the [CODE]config.ini[CODE] file and cannot be modified by the user after first launch.
